# Сравнение ответов: с контролем и без

Один и тот же базовый запрос («Расскажи, что такое нейронная сеть.») отправлен
к DeepSeek дважды — один раз без каких-либо ограничений, второй раз с
явным контролем формата и объёма ответа. Скрипт: [compare_responses.py](compare_responses.py),
сырой вывод: [output_raw.txt](output_raw.txt).

## Что добавили во втором запросе

1. **Явное описание формата ответа** — попросили вернуть ровно три
   пронумерованных пункта, каждый одним коротким предложением.
2. **Ограничение длины** — указали в промпте лимит «не более 60 слов» и
   дополнительно передали `max_tokens=120` в самом API-запросе.
3. **Условие завершения ответа** — попросили модель вывести служебное
   слово `КОНЕЦ` после третьего пункта и передали `stop=["КОНЕЦ"]`, чтобы
   API обрезал генерацию по этому стоп-слову.

## Результат

| Параметр                | Без ограничений | С ограничениями |
|-------------------------|-----------------|-----------------|
| Слов                    | ~593            | ~43             |
| Символов                | ~4368           | ~336            |
| Формат                  | Свободный текст с заголовками, списками, аналогиями и выводом | Ровно 3 пронумерованных пункта |
| Соблюдение лимита длины | Нет — развёрнутый ответ на несколько экранов | Да — короткий ответ, уложился в заданный объём |
| Стоп-условие сработало  | —               | Да, ответ завершился после третьего пункта (модель остановилась сама, не дойдя до стоп-слова `КОНЕЦ`, так как уложилась в формат раньше) |

## Вывод

Без ограничений модель выдаёт развёрнутое эссе с аналогиями, списками,
заголовками и выводами — формат и объём полностью на её усмотрение.

Когда же в промпте явно описан формат (нумерованный список из 3 пунктов),
задан лимит длины (в тексте промпта + параметр `max_tokens`) и указано
условие завершения (служебное стоп-слово + параметр `stop`), модель
выдаёт компактный, предсказуемо структурированный ответ, которым гораздо
проще управлять программно.

Таким образом, явный контроль формата и объёма через промпт и параметры
API заметно меняет поведение модели: ответ становится короче примерно в
13 раз и куда более предсказуемым по структуре.
