Ответ модели:
Рекурсия — это способ решения задачи, когда программа вызывает саму себя до тех пор, пока задача не станет настолько простой, что её легко решить сразу.

Пример: вычисление факториала числа.

**Факториал числа n (обозначается n!) — это произведение всех целых чисел от 1 до n включительно.**

Например, факториал числа 5 равен:

$5! = 5 × 4 × 3 × 2 × 1 = 120$

### Рекурсивная функция для вычисления факториала:

```python
def factorial(n):
    if n == 1:  # базовый случай, возвращаем единицу
        return 1
    else:
        return n * factorial(n-1)  # вызываем функцию снова, уменьшая число
```

**Что происходит?**

При вводе `factorial(5)` сначала будет вызвана сама функция с числом 5, затем снова с числом 4, потом 3, 2 и наконец 1. Когда функция дойдет до числа 1, она вернёт единицу. Затем каждая предыдущая вызванная функция последовательно умножит результат и вернет значение обратно:

$$
5 → 5×4→ 5×4×3 → 5×4×3×2 → 5×4×3×2×1 = 120
$$

Таким образом, рекурсия помогает решать сложные задачи, разбивая их на маленькие шаги, каждый из которых решается просто.

Вот простое объяснение.

**Рекурсия** — это когда функция вызывает саму себя.  
Можно представить это как матрёшку: внутри большой матрёшки — такая же, но поменьше, внутри неё — ещё меньше, и так до самой маленькой.

Главное правило: у рекурсии должна быть **«точка остановки»** (условие выхода), иначе она будет вызывать себя бесконечно, как заезженная пластинка, и программа зависнет.

**Короткий пример на Python:**  
Посчитаем факториал числа (5! = 5 × 4 × 3 × 2 × 1).

```python
def factorial(n):
    if n == 1:        # точка остановки
        return 1
    else:
        return n * factorial(n - 1)   # функция вызывает саму себя

print(factorial(5))   # результат: 120
```

Как это работает:
- `factorial(5)` ждёт результат `factorial(4)` и умножает его на 5.
- `factorial(4)` ждёт `factorial(3)` и умножает на 4.
- ... и так до `factorial(1)`, который просто возвращает 1.
- Затем все «матрёшки» раскрываются обратно, и получается 5×4×3×2×1 = 120.

Простыми словами: рекурсия — это функция, которая решает задачу, разбивая её на всё более мелкие такие же задачи, пока не дойдёт до самой простой.

### Короткий пример на Python — факториал числа

Факториал числа `n` (обозначается `n!`) — это произведение всех чисел от 1 до `n`.  
Например: `5! = 5 × 4 × 3 × 2 × 1 = 120`.

```python
def factorial(n):
    # Базовый случай (самая маленькая матрёшка)
    if n == 1:
        return 1
    # Рекурсивный вызов (открываем следующую матрёшку)
    else:
        return n * factorial(n - 1)

print(factorial(5))  # Выведет 120
```

**Как это работает:**  
1. Сначала вызываем `factorial(5)`.  
2. Функция видит: n не равно 1, значит, возвращаем `5 * factorial(4)`.  
   Чтобы узнать `factorial(4)`, она вызывает саму себя с числом 4.  
3. Так продолжается, пока не дойдём до `factorial(1)`.  
4. `factorial(1)` возвращает **1** (базовый случай).  
5. Теперь все вызовы начинают «закрываться»:  
   `factorial(2) = 2 * 1 = 2`  
   `factorial(3) = 3 * 2 = 6`  
   `factorial(4) = 4 * 6 = 24`  
   `factorial(5) = 5 * 24 = 120`.

**Главное правило:** у рекурсии обязательно должна быть точка остановки (базовый случай), иначе она будет вызывать себя бесконечно, как матрёшка без дна.
