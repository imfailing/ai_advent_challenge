# Week 2 / Day 1 — LLM Agent с веб-интерфейсом

Чат-агент на Flask: принимает запросы пользователя, отправляет их в
DeepSeek API, помнит историю диалога в рамках сессии и отображает ответы
с полным рендерингом Markdown и подсветкой синтаксиса кода.

## Структура

```
day 1/
├── agent.py          # LLMAgent — вся логика API и памяти инкапсулирована здесь
├── app.py            # Flask: маршруты, управление сессиями
├── templates/
│   └── index.html    # Веб-интерфейс (чат, Markdown, подсветка кода)
├── requirements.txt
└── README.md
```

## Возможности

- **Агент как отдельная сущность** — `LLMAgent` инкапсулирует клиент API,
  системный промпт и историю диалога. Flask-приложение знает только метод
  `ask()` и ничего не знает о деталях API.
- **Память сессии** — история всех ходов диалога передаётся в каждый
  следующий запрос, модель помнит контекст разговора. Каждый браузер
  получает свой независимый экземпляр агента.
- **Сброс истории** — кнопка «🗑 Очистить» сбрасывает контекст и начинает
  разговор заново.
- **Markdown-рендеринг** — ответы агента отображаются с заголовками,
  списками, таблицами, цитатами, инлайн-кодом и блоками кода.
- **Подсветка синтаксиса** — блоки кода подсвечиваются через highlight.js
  (Python, JavaScript, Bash, SQL и др.).
- **Метаданные под каждым ответом** — номер хода, название модели, токены
  (вход → выход) и время ответа.

## Установка

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

## Настройка

```bash
export DEEPSEEK_API_KEY="ваш_ключ"
```

## Запуск

```bash
source venv/bin/activate
python app.py
```

Откройте в браузере: [http://localhost:5000](http://localhost:5000)

**Enter** — отправить запрос, **Shift+Enter** — перенос строки.

## API-маршруты

| Метод | Путь    | Описание                                    |
|-------|---------|---------------------------------------------|
| GET   | `/`     | Веб-интерфейс чата                          |
| POST  | `/ask`  | Отправить запрос агенту, получить ответ JSON |
| POST  | `/clear`| Сбросить историю диалога текущей сессии     |
