# Week 2 / Day 2 — Агент с постоянной памятью (SQLite)

Агент сохраняет историю диалога в SQLite. После перезапуска сервера диалог
продолжается с того места, где остановился — модель помнит весь предыдущий
контекст.

## Структура

```
day 2/
├── database.py       # Слой SQLite: init, save, load, clear, list sessions
├── agent.py          # LLMAgent — загружает историю из БД при создании
├── app.py            # Flask: маршруты, сессии, /sessions endpoint
├── templates/
│   └── index.html    # Чат: восстанавливает историю при открытии страницы
├── requirements.txt
└── README.md
```

## Как работает память

```
Первый запуск          Перезапуск сервера
────────────           ──────────────────
LLMAgent(sid)          LLMAgent(sid)
  └─ history = []        └─ history = load_history(sid)  ← из SQLite
  └─ ask("...")          └─ ask("...")  ← модель помнит контекст
       └─ save_message()
```

**Схема БД:**

```sql
sessions (id, created_at, updated_at)
messages (id, session_id, role, content, created_at)
```

## Отличия от Day 1

| Фича                         | Day 1         | Day 2              |
|------------------------------|---------------|--------------------|
| Хранение истории             | RAM (теряется при рестарте) | SQLite (персистентно) |
| Восстановление после рестарта | ❌ Нет        | ✅ Да              |
| Восстановление чата в браузере| ❌ Нет        | ✅ История видна сразу при открытии |
| Просмотр всех сессий         | ❌ Нет        | ✅ GET /sessions    |

## Установка

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

## Настройка

```bash
export DEEPSEEK_API_KEY="ваш_ключ"
```

## Запуск

```bash
source venv/bin/activate
python app.py
```

Откройте [http://localhost:5000](http://localhost:5000).
История прошлых сессий загрузится автоматически — агент продолжит диалог.

## API-маршруты

| Метод | Путь        | Описание                                        |
|-------|-------------|-------------------------------------------------|
| GET   | `/`         | Чат (история восстанавливается из БД)           |
| POST  | `/ask`      | Отправить запрос агенту                         |
| POST  | `/clear`    | Очистить историю текущей сессии                 |
| GET   | `/sessions` | Список всех сессий с количеством сообщений      |
