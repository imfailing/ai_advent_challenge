# Week 2 / Day 4 — Управление контекстом: сжатие истории

Агент автоматически сжимает старые сообщения в суммари и передаёт в каждый
запрос только компактное резюме + последние N сырых сообщений. Это позволяет
вести бесконечно длинный диалог без роста стоимости и без риска переполнения
контекста.

## Как работает компрессия

```
Все сообщения сессии:
  [1..10] [11..20] [21..30] [31..40] ... [последние 10]

После накопления > 20 необработанных:
  [суммари 1..10] [суммари 11..20] ... [последние 10 сырых]

В каждый запрос к LLM уходит:
  system_prompt
  + Краткое содержание предыдущего диалога (суммари)
  + последние 10 сообщений как есть
```

**Параметры (в `agent.py`):**
- `RECENT_KEEP = 10` — сколько последних сообщений не сжимать
- `SUMMARY_BATCH = 10` — сколько сообщений объединять в одно суммари

Сжатие срабатывает автоматически перед каждым запросом: если несжатых
сообщений больше `RECENT_KEEP + SUMMARY_BATCH`, старейшая пачка уходит в
суммари через отдельный вызов LLM.

## Что видно в интерфейсе

Правая панель содержит карточку **🗜 Компрессия контекста**:

| Поле | Значение |
|---|---|
| Всего сообщений | накоплено в истории |
| Сжато в суммари | сколько старых сообщений заменено |
| Блоков суммари | количество сжатых пачек |
| Сырых в запросе | сколько передано в этот запрос без сжатия |

Полоса отображает соотношение суммари / сырых относительно общего числа сообщений.

При появлении нового суммари в чате появляется уведомление-пилюля
«🗜 сжато N сообщений → M блока суммари».

## Структура

```
day 4/
├── models.py         # Реестр моделей (deepseek-v4-flash, deepseek-v4-pro)
├── database.py       # SQLite: messages + context_files + summaries
├── file_parser.py    # Извлечение текста из файлов (текст + PDF)
├── agent.py          # LLMAgent: _maybe_summarize, _create_summary, _build_messages
├── app.py            # Flask: те же маршруты + context в ответе /ask
├── templates/
│   └── index.html    # Карточка компрессии + уведомления суммари в чате
├── requirements.txt
└── README.md
```

## Установка

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

## Настройка

```bash
export DEEPSEEK_API_KEY="ваш_ключ"
```

## Запуск

```bash
source venv/bin/activate
python app.py
```

Откройте [http://localhost:5000](http://localhost:5000).

## API-маршруты

| Метод | Путь | Описание |
|---|---|---|
| GET | `/` | Чат |
| POST | `/ask` | `{message}` → `{answer, usage, session, context{…}}` |
| POST | `/clear` | Сбросить историю и все суммари |
| GET | `/models` | Список моделей |
| GET | `/model` | Активная модель |
| POST | `/model` | Переключить модель |
| POST | `/upload` | Загрузить файл контекста |
| GET | `/context` | Список файлов контекста |
| DELETE | `/context/<id>` | Удалить файл |
| GET | `/sessions` | Все сессии |

### Поле `context` в ответе `/ask`

```json
{
  "total_messages":      42,
  "summaries_count":     3,
  "summarized_messages": 30,
  "raw_in_context":      10
}
```
