# Week 2 / Day 5 — Три стратегии управления контекстом

Агент с переключателем между тремя стратегиями. Каждая определяет,
что именно попадает в запрос к LLM при длинном диалоге.

---

## Стратегия 1 — Sliding Window 🪟

Передаём только **последние N сообщений** (N=10). Всё старше — отбрасывается.

```
История: [1][2][3]...[15][16][17][18][19][20]
В запрос:                  [11][12]...[19][20]
Отброшено: 10 сообщений
```

**Плюсы:** простота, предсказуемый размер контекста.  
**Минусы:** агент «забывает» всё, что было раньше окна.

---

## Стратегия 2 — Sticky Facts 📌

LLM после каждого хода извлекает из диалога **ключевые факты** (цели,
ограничения, решения, договорённости) и сохраняет их в БД. В запрос уходят:
факты в системном промпте + последние N сообщений.

```
System:
  [базовый промпт]
  ## Важные факты
  - цель проекта: мобильное приложение для доставки
  - бюджет: 500K руб
  - дедлайн: сентябрь 2025

Messages: [последние 10 сообщений]
```

Факты извлекаются отдельным вызовом LLM после каждого ответа.

**Плюсы:** агент «помнит» суть даже при длинном диалоге; факты обновляются.  
**Минусы:** дополнительный вызов API на каждый ход; качество зависит от LLM.

---

## Стратегия 3 — Branching 🌿

Диалог можно **разветвить** от любой точки основной ветки. Каждая ветка
продолжается независимо. В запрос уходит полная история текущей ветки
(основная до точки форка + сообщения ветки).

```
Основная:  [A1][A2][A3][A4]
                         └─── Ветка «Вариант А»: [A1][A2][A3][A4][B1][B2]
                         └─── Ветка «Вариант Б»: [A1][A2][A3][A4][C1][C2]
```

Ветки создаются в UI (поле ввода + кнопка «🌿 Создать»).
Переключение между ветками — кликом в списке.

**Плюсы:** можно исследовать разные направления без потери истории.  
**Минусы:** полная история растёт — при длинных ветках стоимость возрастает.

## Новые API-маршруты

| Метод | Путь | Описание |
|---|---|---|
| GET | `/strategy` | Текущая стратегия и ветка |
| POST | `/strategy` | Переключить `{"strategy": "sliding_window\|sticky_facts\|branching"}` |
| GET | `/branches` | Список веток сессии |
| POST | `/branches` | Создать ветку `{"name": "..."}` (только из основной) |
| POST | `/branches/switch` | Переключить ветку `{"branch_id": N}` (null = основная) |
| GET | `/facts` | Текущие факты сессии `{key: value}` |

### Поле `context` в ответе `/ask`

```json
{
  "strategy": "sticky_facts",
  "total_messages": 14,
  "messages_in_context": 10,
  "dropped_messages": 4,
  "facts": {"цель проекта": "...", "бюджет": "..."},
  "branch_id": null,
  "branch_name": "Основная ветка"
}
```
