# Week 3 / Day 2 — Персонализация (профиль пользователя)

Персонализация **поверх модели памяти** из day 1. Агент адаптирует ответы
под профиль пользователя: стиль, формат, уровень и ограничения — автоматически,
без подсказок в каждом сообщении.

---

## Профиль пользователя

Структурированный профиль с явными полями (таблица `profiles`):

| Поле | Значения | Влияние на ответ |
|---|---|---|
| `name` | текст | обращение по имени |
| `role` | текст | контекст («senior backend», «студент») |
| `expertise` | новичок / средний / эксперт | глубина, наличие пояснений терминов |
| `tone` | дружелюбный / нейтральный / формальный | тональность |
| `verbosity` | кратко / средне / подробно | длина ответа |
| `answer_format` | текст | «с примерами кода», «списками», «с аналогиями» |
| `language` | ru / en | язык ответа |
| `constraints` | текст | «без жаргона», «не длиннее 5 предложений» |

Можно держать **несколько профилей** и переключаться между ними — это позволяет
сравнивать поведение агента для разных персон. Первый созданный профиль
становится активным автоматически.

---

## Подключение к каждому запросу

В `agent._make_system()` активный профиль превращается в явные инструкции и
кладётся **первым** в системный промпт (до памяти), чтобы максимально влиять
на стиль:

```
## Профиль пользователя — адаптируй ответ под него
- Имя: Сергей
- Роль: senior backend-разработчик
- Пользователь — эксперт: используй точную терминологию, не объясняй базовое…
- Отвечай развёрнуто, с деталями и пояснениями.
- Тон — строго формальный.
- Формат ответа: с техническими деталями и примерами
- Ограничения: фокус на нюансах и best practices

## Долговременная память … ## Рабочая память …   ← слои из day 1
```

## Проверка: один вопрос, разные профили

`python test_profiles.py` — вопрос **«Что такое REST API?»** от двух персон:

**🟢 Новичок** (новичок, дружелюбный, кратко, без жаргона, ≤5 предложений):
> REST API — это как официант в ресторане: твоя программа говорит ему,
> что нужно, а он передаёт заказ серверу и приносит ответ… *(300 токенов)*

**🔵 Backend-эксперт** (эксперт, формальный, подробно, фокус на нюансах):
> REST API — это архитектурный стиль по Филдингу. Важны не базовые определения,
> а нюансы: идемпотентность, HATEOAS, content negotiation vs versioning,
> RFC 7807, antipatterns… *(с примерами кода, 1447 токенов)*

Один и тот же вопрос → радикально разные ответы. Профиль учитывается
**автоматически**, в самом вопросе про стиль не сказано ни слова.

---

## Что ассистент учитывает автоматически

Помимо ручной настройки, роутер памяти (`_route_memory`) ловит **явно
высказанные** предпочтения прямо из диалога и обновляет активный профиль:

```
Пользователь: «Отвечай мне всегда очень кратко и в дружелюбном тоне.»
   → автонастройка профиля: tone → дружелюбный, verbosity → кратко
```

В UI это показывается плашкой «🎭 автонастройка профиля». Со следующего хода
ответы уже идут в новом стиле. Можно отключить тумблером авто-маршрутизации.

---

## API-маршруты

| Метод | Путь | Описание |
|---|---|---|
| GET | `/profiles` | Список профилей, активный id, поля и enum-значения |
| POST | `/profiles` | Создать профиль `{name, role, expertise, tone, …}` |
| PUT | `/profiles/<id>` | Обновить поля профиля |
| DELETE | `/profiles/<id>` | Удалить профиль |
| POST | `/profiles/activate` | Сделать активным `{profile_id: id\|null}` |
| POST | `/ask` | Запрос; ответ содержит `profile` (активный профиль) |
| POST | `/memory/auto-route` | Вкл/выкл авто-настройку |
