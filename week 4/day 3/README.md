# Week 4 / Day 3 — MCP-инструмент с периодическим выполнением

MCP-сервер с **фоновым планировщиком 24/7**: он периодически собирает данные
в SQLite, обрабатывает отложенные напоминания и сохраняет снапшоты сводок.
Агент по запросу получает **агрегированный результат**.

---

## Что внутри

```
            ┌─────────────────────────── mcp_server.py ───────────────────────────┐
            │  при старте → Scheduler (фоновый поток, 24/7)                        │
app.py ──▶  │     каждый тик: collector.sample() → store (SQLite)                  │
agent.py    │                 store.fire_due_reminders()                           │
(DeepSeek)  │                 каждые N тиков: snapshot сводки → store              │
            │  MCP-инструменты: summary / recent_summaries / add_reminder /        │
            │                   due_reminders / pending_reminders / status         │
            └─────────────────────────────────────────────────────────────────────┘
```

Инструмент выполняет все три требования:
- **сохраняет данные** → SQLite (`store.py`: samples / reminders / summaries);
- **выполняется по расписанию** → фоновый поток `Scheduler` (`scheduler.py`);
- **возвращает агрегированный результат** → `summary()` (count, avg, min, max, last).

---

## Инструменты MCP

| Инструмент | Параметры | Назначение |
|---|---|---|
| `summary` | `minutes` | агрегат метрики за окно (count, avg, min, max, last) |
| `recent_summaries` | `limit` | последние периодические снапшоты сводок |
| `add_reminder` | `text, in_seconds` | **отложенное** напоминание (сработает через N сек) |
| `due_reminders` | — | напоминания, которые уже сработали |
| `pending_reminders` | — | напоминания, ещё ожидающие срока |
| `scheduler_status` | — | статус планировщика (интервал, тики, всего измерений) |

## Запуск

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
export DEEPSEEK_API_KEY="ваш_ключ"

python app.py            # подождать сбор → попросить агента сводку
python app.py "Дай сводку за 5 минут"
```

Пример (агент вызывает MCP-инструменты и использует результат):

```
❓ Дай сводку по метрике за последние 5 минут
   🔧 MCP summary({'minutes': 5}) → {count: 11, avg: 55.27, min: 42.8, max: 64.2…}
🤖 Сводка за 5 минут (active_users): измерений 11, среднее 55.27,
   мин 42.8, макс 64.2, последнее 54.5.

❓ Поставь напоминание «снять метрики» через 2 секунды
   🔧 MCP add_reminder({'text':'снять метрики','in_seconds':2}) → {id:1, due_at:…}
🤖 ✅ Напоминание поставлено — сработает через 2 секунды.
```

Интервал сбора и частоту снапшотов можно задать переменными окружения:
`SCHED_INTERVAL` (сек, по умолчанию 1.0), `SCHED_SUMMARY_EVERY` (тиков, 10).

> Примечание: MCP запускает сервер подпроцессом и наследует только базовое
> окружение, поэтому эти переменные надо передавать самому серверу
> (либо через `StdioServerParameters(env=…)`).
