- ✅ **Модель запускается локально** — Ollama-сервер отвечает на `/api/version`: `{"version":"0.31.2"}`
- ✅ **CLI** — `ollama run qwen2.5:1.5b "сколько будет 2+2?"` → `4`
- ✅ **HTTP API** — `POST http://localhost:11434/api/chat` возвращает ответ

## 3 запроса разной сложности (через HTTP API)

### [1/3] Простой факт
```
❓ Столица Франции — назови одним словом.
🤖 Париж.
   [44→5 токенов · 0.2 с]
```
✅ Верно, мгновенно.

### [2/3] Рассуждение / логика (задача-ловушка)
```
❓ В комнате 3 кота. Каждый кот видит 2 других котов. Сколько всего котов?
🤖 …Таким образом, в комнате всего 4 кота.
   [70→282 токена · 1.88 с]
```
⚠️ **Неверно** (правильный ответ — 3). Модель 1.5B запуталась в рассуждении и
насчитала 4. Это честная демонстрация ограничений маленькой локальной модели
на логических задачах-ловушках.

### [3/3] Генерация кода
```
❓ Напиши функцию на Python: список простых чисел до N. Только код.
🤖 def prime_numbers(n):
       primes = []
       for num in range(2, n + 1):
           if all(num % i != 0 for i in range(2, int(num**0.5) + 1)):
               primes.append(num)
       return primes
   [55→81 токен · 0.62 с]
```
✅ Корректный рабочий код (проверка делимости до √num).

## Вывод

Локальная LLM запущена и отвечает на запросы через CLI и HTTP API. Скорость
высокая (0.2–1.9 с на запрос, полностью офлайн, без API-ключей). Простой факт и
генерация кода — корректны; на логической задаче-ловушке модель 1.5B ошиблась,
что ожидаемо для её размера — для сложного рассуждения нужна модель побольше
(например `qwen2.5:7b` или `llama3.1:8b`).
