# Week 6 / Day 3 — RAG полностью на локальной модели

RAG-пайплайн из недели 5, подключённый к **локальной LLM**: retrieval
выполняется локально, генерация — через локальную модель (Ollama). Плюс
сравнение с облачной моделью (DeepSeek) на одном и том же пайплайне.

---

## Что где выполняется

```
вопрос
  │  RETRIEVAL — всегда ЛОКАЛЬНО:
  │    embed (fastembed, локально) → поиск по SQLite-индексу → rerank (cross-encoder локально)
  ▼
контекст + вопрос
  │  ГЕНЕРАЦИЯ — переключаемый бэкенд:
  │    backend="local" → qwen2.5:1.5b через Ollama (офлайн, без ключей)
  │    backend="cloud" → deepseek-v4-flash (для сравнения)
  ▼
ответ + источники
```

Индекс — из недели 5 (`loader/chunking/embedder/index_store`, `corpus/`
скопированы сюда; `build_index.py structural`). Эмбеддинги и реранкер —
локальные модели fastembed, поэтому **retrieval не ходит в облако**.

Чисто локальный путь (`backend="local"`) не требует ни ключей, ни сети.

---

## Сравнение local vs cloud (`compare.py`, 10 вопросов)

Retrieval одинаковый и локальный; отличается только генератор.

| Бэкенд | Верных | Ср. hit | Ген., с | Макс, с | Разброс | Пустых |
|---|---|---|---|---|---|---|
| **local** (qwen2.5:1.5b) | 9/10 | 88% | **1.48** | 3.87 | 0.88 | 0 |
| **cloud** (deepseek-v4-flash) | 10/10 | 100% | 3.56 | 5.74 | 1.23 | 0 |

### Оценка

- **Качество:** облако чуть выше (10/10 против 9/10). Локальная 1.5B провалила
  только «списковый» вопрос про три стратегии — остальные 9 верны.
- **Скорость:** локальная генерация **быстрее** (~1.5 с против ~3.6 с) — крошечная
  модель без сетевой задержки. Retrieval общий (~0.1 с после прогрева).
- **Стабильность:** обе — 0 пустых ответов; у локальной меньше разброс времени
  (0.88 против 1.23 с).

**Вывод:** RAG-система работает **полностью локально** с качеством 9/10 и
скоростью выше облака. Облако выигрывает точность на сложных/списковых вопросах,
но требует ключа и сети. Для приватных/офлайн-сценариев локальный путь —
рабочий, а разрыв в качестве закрывается моделью побольше (7b+).

## Запуск

```bash
# нужен Ollama с моделью (week 6 / day 1): ollama pull qwen2.5:1.5b
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt
python build_index.py structural       # индекс (из недели 5)

python compare.py local                # только локально (офлайн)
DEEPSEEK_API_KEY=... python compare.py # local + cloud сравнение
```
