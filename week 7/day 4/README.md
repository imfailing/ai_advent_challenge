# Week 7 / Day 4 — Ассистент, работающий с файлами

Ассистент, который **реально оперирует файлами**, а не просто отвечает текстом.
Вы задаёте **цель** («найди все использования X и собери отчёт»), а агент сам
решает, какие файлы прочитать/искать/проанализировать и что создать/изменить —
через файловые инструменты MCP.

Все операции — в песочнице `workspace/` (агент не выходит за её пределы).

---

## Что умеет (через MCP `fs_mcp_server.py`)

| Инструмент | Действие |
|---|---|
| `list_files(subdir)` | список файлов проекта |
| `read_file(path)` | чтение файла |
| `search(pattern, glob)` | поиск (regex) по многим файлам → `file:line:текст` |
| `write_file(path, content)` | создать / изменить файл |

Агент (`agent.py`, DeepSeek tool-calling) получает **цель** и сам ведёт работу:
сначала собирает данные (search/read по нескольким файлам), потом пишет результат.

---

## Реализованные сценарии (`run.py`)

**1. Найти все места использования компонента** (`Logger`):
агент → `list_files` → `search("Logger")` → `read_file` ×4 → `write_file
docs/logger_usage.md` (импорты, создание экземпляров, вызовы методов — таблицами).

**2. Проверить соответствие правилу/инварианту** (докстринг модуля в каждом .py):
агент → `list_files` → `read_file` ×5 → анализ → `write_file compliance_report.md`
со списком нарушителей. Корректно находит `config.py` и `utils.py` (без докстринга).

Оба сценария: агент работает с **5+ файлами**, создаёт файл-результат.

---

## Запуск

```bash
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt
export DEEPSEEK_API_KEY="ваш_ключ"

python run.py            # оба сценария (с пересозданием песочницы и diff)
python run.py 1          # только сценарий 1
python run.py "своя цель на уровне цели, а не 'открой файл X'"
```

`run.py` пересоздаёт `workspace/`, снимает снимок ДО, запускает агента и
показывает **вызванные инструменты** и **unified diff** созданных/изменённых
файлов. Изменения при этом **сохраняются** в песочнице.
