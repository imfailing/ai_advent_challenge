# Week 7 / Day 5 — AI-дайджест изменений репозитория

Реальное AI-приложение под конкретную задачу, которую я действительно хочу
автоматизировать.

---

## Какую задачу решаю

Я коммичу в этот репозиторий каждый день. Вручную писать понятный **дайджест
изменений** (что нового за неделю, release notes) — рутина: надо просмотреть
десятки коммитов, сгруппировать, переписать по-человечески.

Сервис делает это сам: читает git-историю и превращает сырые коммиты в
аккуратный Markdown-дайджест, а затем (опционально) публикует его в
Telegram-канал.

## Как участвует AI

Ключевой шаг — **AI (DeepSeek)** превращает машинные сообщения коммитов
(`week7/day4: file-operating assistant …`) в человекочитаемые release notes:
- **группирует** изменения по типам (новое / исправления / документация / прочее);
- **выделяет главное** — короткое резюме периода;
- **переписывает** технические сабджекты в понятный, дружелюбный язык,
  убирая хэши и префиксы вроде `week7/day4:`.

```
git log (реальная история)
   │
   ▼
DeepSeek: группировка + резюме + переписывание   ← здесь работает AI
   │
   ▼
digest.md  ──(опц.)──▶  Telegram-канал
```

## Реальное окружение и «может быть не полностью рабочим»

- Работает на **настоящей** git-истории репозитория (`gitlog.py` → `git log`).
- Публикация в Telegram (`notify.py`) — реальная интеграция через Bot API, но
  требует `TELEGRAM_BOT_TOKEN` и `TELEGRAM_CHAT_ID`. **Без них сервис не падает** —
  шаг публикации возвращает `skipped`, дайджест остаётся в файле. Это и есть
  «не полностью рабочее из-за доступов», но пайплайн целостный.

## Запуск

```bash
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt
export DEEPSEEK_API_KEY="ваш_ключ"

python run.py --last 12                       # по последним N коммитам
python run.py --since "1 week ago"            # за период
python run.py --range af1211e..HEAD           # по диапазону
python run.py --since "7 days ago" --notify   # + публикация в Telegram

# публикация (опционально):
export TELEGRAM_BOT_TOKEN="..."; export TELEGRAM_CHAT_ID="..."
```

Дайджест сохраняется в `digest.md` и печатается в консоль.

### Автоматизация (GitHub Action)

`.github/workflows/weekly-digest.yml` (в корне репо) запускается **раз в неделю**
(и по кнопке): собирает коммиты за 7 дней, генерирует дайджест, кладёт артефактом
и — если заданы секреты Telegram — публикует. Секреты: `DEEPSEEK_API_KEY`
(обяз.), `TELEGRAM_BOT_TOKEN`/`TELEGRAM_CHAT_ID` (опц.).
